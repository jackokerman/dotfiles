# Review GodspeedJS public publishing and API-use boundaries

## Plan

Before publishing GodspeedJS publicly or submitting any public integration, review the current Godspeed API docs, EULA, privacy policy, and developer/API terms if present.

Scope:

- confirm whether broad task-management clients are acceptable API use;
- decide whether undocumented endpoints used for parity need Godspeed support confirmation before public behavior;
- keep the project clearly unofficial;
- avoid copied logos, screenshots, docs, or brand assets without permission;
- document the privacy posture: token stays local, requests go only to Godspeed, and no telemetry exists unless deliberately added later.

This follow-up does not block the private local workflow.
